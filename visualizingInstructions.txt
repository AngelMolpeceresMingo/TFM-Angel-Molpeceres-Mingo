La estructura típica de las carpetas es esta:

|—— <SeqName>
     |—— img1
         |—— 000001.jpg
         |—— 000002.jpg
         |—— ...
     |—— gt
         |—— gt.txt (opcional, si tienes datos de ground truth)
     |—— det
         |—— det.txt (opcional, detecciones públicas)
         

### **2. Configurar el script de visualización**
El repositorio incluye un script llamado `MOT/MOTVisualization.py`, que permite generar visualizaciones. Para configurarlo:

1. **Ajusta los parámetros en la clase `MOTVisualizer`:**
   - `seqName`: Nombre de la secuencia que deseas visualizar.
   - `FilePath`: Ruta del archivo de datos (por ejemplo, resultados de seguimiento).
   - `image_dir`: Directorio que contiene las imágenes del clip.
   - `mode`: Define el tipo de visualización:
     - `None`: Para visualizar los resultados del método.
     - `raw`: Solo video sin anotaciones.
     - `gt`: Para mostrar las anotaciones del *ground truth*.

2. **Configura la función `generateVideo`:**
   - Define opciones como:
     - `displayTime`: Mostrar el número de fotograma (True/False).
     - `displayName`: Nombre del método.
     - `showOccluder`: Mostrar objetos ocluidos en los datos del *ground truth*.
     - `fps`: Velocidad de fotogramas para el video generado.

Ejemplo en código:
```python
visualizer = MOTVisualizer(
    seqName="MOT17-04",
    FilePath="./res/MOT17-04.txt",
    image_dir="./data/MOT17-04/img1",
    mode="gt",
    output_dir="./vid"
)
visualizer.generateVideo(displayTime=True, displayName="MyTracker", showOccluder=False, fps=30)
```

---

### **3. Ejecutar el script**
Corre el script desde la terminal o un entorno Python:
```bash
python MOT/MOTVisualization.py
```

Esto generará un video con las visualizaciones en la carpeta especificada (`output_dir`).
